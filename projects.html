Tile Map Editor(C++/SDL)

A Tile Map Editor that includes tile deletion when placing a new tile, tile map saving as a text document, tile importing, and map importing. The map size is fixed to 40 by 25 and tile width and height of 48 which results in a 1920 by 1200 width and height.

UPDATE: Recently updated the tile editor. Added width and height adjustments so I can size the map. Added zoom in and zoom out effect for the tile map. Fixed the tile class; each tile no longer takes and loads an image, images are now pre-loaded on start-up which reduces load times significantly and allows for much larger maps. Also took out the pre-loaded tile sheet and loaded 2 default tiles and more can be loaded in with the import tile function. Link to the old tile editor video.

Zero Gravity(C++/SDL)

This was my first program made with C++ and SDL. This game was done in one day after hearing about several 1-day space shooter challenges I decided to see what I could do with the 1-day constraint and limited knowledge of C++ and SDL. The particle engine is homemade and is also used to spawn white particles to act as a parallax scrolling field of stars. This game also includes a homemade Vector class that includes operator overloading and other operations like finding the Magnitude of a Vector to know what distance it has traveled from it's original spawn point - I use that to delete the particles after they've traveled a certain distance from their spawn point. The main menu is also only keyboard controlled.

MMO-JRPG(C++/SDL/enet)

Work In Progress

This is an MMO-JRPG using enet and my Tile Map Editor. This game has a turn-based Final Fantasy-esque battle system with a custom magic system unlike that of Final Fantasy. The magic system has the user select 2 spells to create a powerful spell that is cast on the opponent. Aside from gameplay I have a custom simple save encryption file which uses a key to encrypt and decrypt local save files. The encryption key changes every time the game saves, and the key is saved locally for the time being. The game's server has the authority in the game world which means when a player moves the server checks if the move is valid(i.e if the user hasn't disconnected or is not hitting a wall/object), if that is not the case the player may move. Client side if the player has already moved, and if the move is safe it will remain where it moved, but will only update for server if the move was valid. Also implemented is a quest system that reads quests off of an XML sheet and an inventory system with navigation as well as function to use items in the inventory.

My First Neural Network(python)

This is my first Neural Network and introduction to AI! I'm using the numpy library in python to create matrices and compute math with the matrices. This neural network uses the Sigmoid activation function. I chose the Sigmoid activation because I only need an output between 0 and 1 since I'm only using binary matrices and I'm not doing anything overly complicated yet. I feed the neural network 4 1x4 matrices. If the matrix has a 1 in the first column ([1,0,0,0]) then the bots answer should be as close to 1 as possible, but if the matrix has a 0 in the first column ([0,1,0,1]) then the bots answer should be as close to 0 as possible. The neural network trains off these 4 examples 60,000 times using matrix multiplication coupled with forward propagation, backpropagation, and gradient descent to fix the error margin so the bot's answer is closer to the right answer, and showing the error margin after intervals of 10,000 so you can see how effective the training has been. Once training is done I have 12 4x1 matrices and the bot picks 1 at random, gives an expected answer, and then gives the bot answer. If the answer is greater than 0.5 a cat picture will be shown, otherwise a dog picture will be shown.
